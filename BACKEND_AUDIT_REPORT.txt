[C1] Stream retry produces DUPLICATE content
  File: /mnt/c/Users/24345/Desktop/novel-continuation-main/backend/services/llm_client.py
  Lines: 197-211
  Issue: chat_completion_stream() is an async generator. The retry loop at line
         201 re-invokes do_request() on failure, which starts a NEW stream from
         the beginning. Any chunks already yielded to the caller (line 202:
         "yield text") are NOT un-done. The caller accumulates these into
         full_text (continuation_service.py:90), resulting in DOUBLED content
         on retry.
  Impact: Generated chapters may contain the first half duplicated on network
          glitches. Silent data corruption.
  Fix:   Buffer all chunks in the generator; only yield after stream completes
         successfully. On retry, don't yield partial results:

         async def do_request():
             ...
         collected = []
         for attempt in range(MAX_RETRIES + 1):
             try:
                 async for text in do_request():
                     collected.append(text)
                 for t in collected:
                     yield t
                 return
             except ...:
                 collected.clear()  # discard partial
                 if attempt < MAX_RETRIES:
                     await asyncio.sleep(...)
                 else:
                     raise

[H2] preprocess_service.py uses text.index() instead of regex offset
  File: /mnt/c/Users/24345/Desktop/novel-continuation-main/backend/services/preprocess_service.py
  Lines: 70, 88, 103, 110
  Issue: After regex matching within a substring (text[start:end]), the code
         uses text.index(ch_match.group(), start) to find the absolute
         position. If the matched text appears multiple times, index() returns
         the FIRST occurrence after start, which may NOT be the actual match
         position. This can cause incorrect outline/chapter text splitting.
  Example: If chapter title "第一章 开端" appears in both the outline and
           the chapter body, index() picks the wrong one.
  Fix:   Use regex offset directly:
         Line 70: ch_abs = start + ch_match.start()
         Line 88: ch_abs = start + ch_match.start()
         Line 103: ch_pos = first_ch.start()  (since searching full text)
         Line 110: ch_pos = first_ch.start()

  File: /mnt/c/Users/24345/Desktop/novel-continuation-main/backend/services/llm_client.py
  Lines: 16-21
