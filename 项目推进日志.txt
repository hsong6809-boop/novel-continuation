一、今日完成工作
----------------

1. 后端Service层重构（6个模块）
   - llm_client.py：通用LLM调用客户端，支持多provider切换

2. 设置页修复
   - 后端：保存时自动同步active_model与provider的default_model
   - 前端SettingsPage：保存前同步active_model

日期：2026-04-28
状态：分卷大纲功能开发完成

一、今日完成工作
----------------

1. 数据库层
   - 新增 volume_outlines 表（分卷大纲存储）
   - chapter_outlines 表新增 volume_id 字段（关联分卷）

2. 后端新增
   - services/volume_outline_service.py：分卷大纲 CRUD + AI 生成
   - routers/projects.py 新增路由：
     * GET    /projects/{id}/outlines/volumes          分卷列表
     * POST   /projects/{id}/outlines/volumes          创建分卷
     * GET    /projects/{id}/outlines/volumes/{vid}    获取单卷
     * PUT    /projects/{id}/outlines/volumes/{vid}    更新分卷
     * DELETE /projects/{id}/outlines/volumes/{vid}    删除分卷
     * POST   /projects/{id}/outlines/volumes/generate AI 批量生成分卷
     * POST   /projects/{id}/outlines/chapters/batch-generate  按卷批量生成章纲

3. 后端修改
   - services/outline_service.py：新增 batch_generate_outlines_for_volume()
   - services/overall_outline_service.py：总纲生成 prompt 去掉卷规划，只保留整体结构

4. 前端 API
   - frontend/src/api/client.js 新增 8 个函数：
     listVolumeOutlines, getVolumeOutline, createVolumeOutline,
     updateVolumeOutline, deleteVolumeOutline, generateVolumeOutlines,
     batchGenerateOutlines

5. 前端组件
   - OutlinePanel.jsx 三栏重构：
     * 总纲编辑器：只管整体故事结构
     * 分卷大纲编辑器：AI 一键生成 + 手动增删改 + 按卷设置章节范围
     * 逐章章纲：按卷分组展示，每卷可批量 AI 生成，支持单章编辑

6. 构建验证
   - 后端 Python 语法检查全部通过（5个文件）
   - 前端 vite build 通过（3.09s，386KB gzip 113KB）

二、架构设计变更
----------------

- 总纲只管整体故事结构，不再混入分卷
- 分卷大纲独立管理，作为总纲和逐章章纲的中间层
- 逐章章纲通过 volume_id 关联分卷，按卷分组展示
